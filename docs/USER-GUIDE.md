# Jira 통합 관리 도구 - 사용자 가이드

> 이 문서는 배포된 서비스를 사용하는 사람을 위한 가이드입니다.
> 프로젝트 설치 및 커스터마이징은 [SETUP.md](./SETUP.md)를 참고하세요.

---

## 전체 흐름

```
1. 초기 설정 (최초 1회)
   사용자 등록 → 팀 생성 → 프로젝트 등록 → 동기화 프로필 설정 → 팀에 연결

2. 일상 사용
   사용자 선택 → 동기화 실행
```

---

## 1. 초기 설정

서비스를 처음 사용할 때 아래 순서대로 설정합니다.
설정 페이지는 메인 화면 우측 상단 ⚙️ 버튼으로 진입합니다.

### 1-1. 사용자 등록 (`설정 > 사용자 관리`)

팀원 각자의 Jira 인증 정보를 등록합니다.

| 항목 | 설명 |
|------|------|
| 이름 | 사용자 표시명 |
| 팀 | 소속 팀 (팀이 먼저 생성되어 있어야 함) |
| Ignite Jira 이메일 / API 토큰 | 소스 Jira 인스턴스 인증 정보 |
| HMG Jira 이메일 / API 토큰 | 대상 Jira 인스턴스 인증 정보 |
| HMG 사번 | HMG 사용자 식별 ID |

**등록 절차:**
1. "사용자 추가" 클릭
2. 이름, 팀 입력
3. 각 Jira 인스턴스의 이메일 + API 토큰 입력 후 **"인증 확인"** 클릭
4. 두 인스턴스 모두 인증 성공 시 저장 가능

> **Jira API 토큰 발급:** [Atlassian API 토큰 관리](https://id.atlassian.com/manage-profile/security/api-tokens) 에서 생성

### 1-2. 팀 생성 (`설정 > 팀 관리`)

동기화 단위가 되는 팀을 생성하고, 소스/대상 프로젝트를 연결합니다.

**설정 항목:**

| 항목 | 설명 |
|------|------|
| 팀 이름 | 팀 식별용 이름 |
| 소스 프로젝트 | 동기화 기준이 되는 프로젝트 (1개) |
| 대상 프로젝트 | 동기화 대상 프로젝트 (복수 선택 가능) |
| 동기화 프로필 | 각 대상 프로젝트에 적용할 필드 매핑 규칙 |
| 팀원 | 이 팀에 소속될 사용자 |

**설정 절차:**
1. "팀 추가" 클릭 → 팀 이름 입력
2. 편집 모드에서:
   - ⭐ 소스 프로젝트 선택
   - 📁 대상 프로젝트 선택 (토글)
   - 각 대상에 동기화 프로필 지정
   - 👤 팀원 지정

### 1-3. 프로젝트 등록 (`설정 > 프로젝트 관리`)

Jira 프로젝트를 시스템에 등록합니다.

**등록 절차:**
1. "프로젝트 추가" 클릭
2. Jira 인스턴스 선택 (Ignite / HMG)
3. 프로젝트 키 입력 후 **"확인"** 클릭 (Jira에서 존재 여부 검증)
4. 검증 성공 시 프로젝트명, 보드 ID 자동 조회
5. 연결할 팀 선택 후 저장

### 1-4. 동기화 프로필 설정 (`설정 > 동기화 방식 관리`)

소스 → 대상 프로젝트 간 필드 매핑 규칙을 정의합니다.

**설정 항목:**

| 항목 | 설명 |
|------|------|
| 소스/대상 프로젝트 | 매핑 대상 프로젝트 쌍 |
| 필드 매핑 | 소스 필드 → 대상 필드 변환 규칙 |
| 상태 매핑 | 소스 상태 ID → 대상 상태 ID 매핑 |
| 워크플로우 전이 | 대상 프로젝트의 허용된 상태 전이 경로 |
| 에픽 허용 목록 | 동기화 대상 에픽 제한 (선택사항) |

**설정 절차:**
1. 프로필 생성 → 소스/대상 프로젝트 선택
2. 필드 매핑 추가 (소스 필드명 → 대상 필드명 + 변환 로직)
3. 상태 매핑 설정 (소스 상태 → 대상 상태)
4. 워크플로우 전이 규칙 설정 (상태 A → 상태 B 허용 여부)
5. (선택) 에픽 허용 목록으로 동기화 범위 제한

## 2. 일상 사용

### 2-1. 사용자 선택

서비스 접속 시 사용자 선택 화면이 표시됩니다.

1. 본인 이름 검색/선택
2. "시작하기" 클릭
3. Jira 인증 정보가 미완료인 경우 → 인증 설정 화면으로 자동 이동

### 2-2. 동기화 실행

메인 화면에서 동기화를 실행합니다.

**동기화 모드:**

| 모드 | 설명 | 용도 |
|------|------|------|
| **전체** | 소스 프로젝트의 모든 이슈를 동기화 | 정기 전체 동기화 |
| **에픽 지정** | 특정 에픽의 이슈만 동기화 | 에픽 단위 동기화 |
| **티켓 지정** | 특정 티켓 + 연결 이슈만 동기화 | 개별 이슈 동기화 |

**실행 절차:**
1. 동기화 모드 선택
2. (에픽/티켓 지정 시) 대상 ID 입력
3. 동기화 버튼 클릭
4. 실시간 로그로 진행 상황 확인
5. 완료 후 요약 결과 확인 (성공/실패 건수)

### 2-3. 자동 동기화 (Daily 배치)

GitHub Actions를 통해 매일 자동으로 전체 동기화가 실행됩니다.
수동 실행 없이도 데이터가 일일 단위로 동기화됩니다.

> 배치 스케줄 변경은 [SETUP.md - 배치 스케줄 변경](./SETUP.md#7-배치-스케줄을-변경하고-싶을-때)을 참고하세요.

## 3. 설정 변경

### 팀원 추가/제거
`설정 > 팀 관리` → 해당 팀 편집 → 팀원 토글

### 새 프로젝트 동기화 대상 추가
1. `설정 > 프로젝트 관리` → 프로젝트 등록
2. `설정 > 동기화 방식 관리` → 프로필 생성
3. `설정 > 팀 관리` → 대상 프로젝트 추가 + 프로필 지정

### 사용자 전환
메인 화면 상단의 사용자 드롭다운에서 변경 가능

## FAQ

**Q: Jira API 토큰은 어디서 발급하나요?**
A: [https://id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens) 에서 "Create API token"으로 생성합니다.

**Q: 동기화 중 실패한 티켓은 어떻게 되나요?**
A: 실패한 티켓은 로그에 표시되며, 다음 동기화 시 다시 시도됩니다. 개별 실패가 전체 동기화를 중단하지 않습니다.

**Q: 동기화 방향은?**
A: 소스 프로젝트 → 대상 프로젝트 단방향입니다. 역방향 동기화가 필요하면 별도 프로필을 생성해야 합니다.
